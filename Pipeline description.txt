Процес безперервної інтеграції та розгортання (CI/CD) розділений на дві незалежні частини для бекенду та фронтенду:

1. Бекенд (Django + Docker):
Пайплайн реалізовано за допомогою GitHub Actions і запускається автоматично при кожному пуші або створенні pull request у гілку `main`. Він складається з двох етапів (jobs):
Етап Build & Test: Система перевіряє (checkout) актуальний код із репозиторію та збирає Docker-образ застосунку. Далі, всередині ізольованого контейнера запускаються автоматичні тести Django. Для швидкодії та незалежності тестів конфігурація динамічно перемикається на використання in-memory бази даних SQLite (`sqlite://:memory:`).
Етап Deploy: Виконується лише у випадку успішного проходження всіх тестів і лише для гілки `main`. За допомогою `curl` відправляється POST-запит на захищений webhook (Deploy Hook) платформи **Render**. Це дає сигнал серверу підтягнути нові зміни, перезібрати контейнер, застосувати міграції бази даних PostgreSQL та оновити працюючий застосунок без простою (zero-downtime).

2. Фронтенд (React):
Розгортання клієнтської частини налаштовано через вбудований механізм CI/CD платформи Render для статичних сайтів (Static Site). 
При оновленні коду в гілці `main`, Render автоматично завантажує зміни.
Виконується встановлення залежностей та збірка оптимізованого продакшн-білду (команди `npm install && npm run build`).
Зібрані файли автоматично розгортаються на глобальній CDN. Для коректної роботи маршрутизації Single Page Application (SPA) налаштовано правило Rewrite, яке перенаправляє всі запити на `index.html`.
